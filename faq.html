What is Project Telescope?

Project Telescope is a Red Hat sponsored upstream, open source initiative, which will enable customers to automate security governance and compliance within DevSecOps activity against a customer-specific set of technical and organizational requirements, as well as provide risk assessment and management automation and proactive visualization. It functions by providing automated compliance attestation between a product or application hosted on an OpenShift container, and a variety of customer-defined security capabilities, whilst providing visibility into the status of both technical and business requirements via a dynamic, natively-embedded dashboard.

How do I join the Project?

There are two main ways of joining the Project! First, you can reach out to the Project Leads Robert Erenberg-Andersen ( ), Chris Jenkins ( ), or Russell Builta ( ). Alternatively, you can simply head on over to and start contributing!

What's the intended end-goal of the Project?

like any Open Source community, Project Telescope is intended to provide a collaborative space for development of this industry-new capability. Whilst the upstream Project is an end unto itself, we do hope to develop it to the point that it can be consumed as a downstream, official Red Hat Product one day!

What do I need to be able to contribute effectively?

There are no prerequisite requirements for contribution beyond an interest in the Project, and a love for security and automation! That being said, we are primarily seeking the following skillsets: Front-end Developers, Back-end Developers, AppSec SMEs, Site Reliability Engineers, Platform Architects, and Security Engineers
